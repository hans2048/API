# 업무일지 시스템 UI 항목 상세 명세 (Specification)

## 1. 기본 정보 (Basic Information)

| 항목명 | 필드타입 | 필수여부 | 비고 |
| :--- | :--- | :---: | :--- |
| **작성일자** | Date | 필수 | 기본값은 현재 날짜로 설정 |
| **작성자** | String | 필수 | 로그인 세션에서 자동 로드 가능 |
| **부서** | Select | 필수 | 조직도와 연동된 부서 리스트 |

## 2. 업무 상세 테이블 (Work Details)

| 항목명 | 필드타입 | 설명 |
| :--- | :--- | :--- |
| **우선순위** | Enum | 긴급/높음/보통/낮음 중 선택 |
| **업무명/내용** | Text | 수행한 구체적인 태스크 단위 기술 |
| **진행률** | Integer | 0~100 사이의 수치 데이터 |
| **상태** | Select | 진행중, 완료, 보류, 지연으로 구분 |

## 3. 기록 및 공유 (Logs & Issues)
- **특이사항 및 이슈**: 
  - 업무 수행 중 발생한 기술적/커뮤니케이션 문제 기술.
  - 리스크 관리를 위한 데이터로 활용.
- **내일 주요 계획**:
  - 차일 업무의 우선순위를 미리 설정하여 업무 연속성 확보.

## 4. 시스템 기능 요구사항
1. **자동 저장**: 작성 중 브라우저 종료 시를 대비한 임시 저장 기능.
2. **데이터 연동**: 프로젝트 관리 도구(Jira, Trello 등)의 티켓 번호와 연결 가능하도록 설계.
3. **통계**: 주간/월간 단위로 진행률을 시각화하여 대시보드 제공.
